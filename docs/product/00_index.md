# VANAVIL Product Docs Index

Use these files in order if you want to move from planning to implementation.

1. `01_solution_architecture.md`
   Overall system shape, responsibilities, layers, lifecycle rules, deployment topology, and MVP rollout order.

2. `02_firestore_schema_and_indexes.md`
   Firestore collections, field definitions, transition rules, composite indexes, and backend-owned transactions.

3. `03_screen_flow_and_navigation.md`
   Admin and child journeys, route maps, primary navigation, and empty-state guidance.

4. `04_design_system.md`
   Theme direction, palette, components, spacing, motion, and the visual split between admin web and child mobile.

5. `05_starter_module_structure.md`
   Suggested workspace layout, feature modules, repositories, Firebase integration plan, starter code skeletons, and MVP-first implementation order.

6. `06_local_commands.md`
   Day-to-day local run, analysis, and test commands for the Python API, admin web, child app, and Firebase rules tests.

7. `07_current_state_runbook.md`
   Current implementation truth source: active architecture, auth flow, S3 integration, environment variables, troubleshooting notes, and release checklist.

These docs are intentionally MVP-focused: they keep the product scalable without making the first version heavy.
